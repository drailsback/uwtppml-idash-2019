To Run:

(1) Configure TI_Settings with the number of shares of each type needed for the protocol
(2) Configure Party0_Settings with paths to its input/output files
(3) Configure Party1_Settings with paths to its input/output files
	- Namely:
		- x_input_path: input -- secret shares of data set (csv)
		- y_input_path: input -- secret shares of class vector (csv)
		- xor_shares_path: output -- location to store binary shares sent from TI during execution (csv)
		- additive_shares_path: output -- location to store additive shares sent from TI during execution (csv)
		- attribute_count: input -- columns in data set x
		- instance_count: input -- rows in data set x
		- learning_rate: input -- learning rate to use for the LR protocol
		- decimal_precision: input -- number of bits reserved for fixed point decimal values (must be consistent with the way x and y are shared)
		- integer precision: input -- number of bits needed to represent max, min integer values during protocol: Note (decimal_precision + integer_precision) < 32  ( operations on a ring mod 2^64 ) 
		- output_path: output -- location to store the weights (in the clear) generated by the model 
		(csv)

	- NOTE: 
		- x, y must be different files that are correlated inputs pre-generated by a client
		- xor_share_path and additive_shares_path MUST be different for each party
		- output_path MAY be different. The file generated will be the same for both parties
		- all other fields MUST be the same between parties

(4) In idash2019_rust, 'cargo build'
(5) In one terminal, 'cargo run TI_Settings' Note: -- must be running before parties
(6) In a separate terminal, "cargo run Party0_Settings"
(7) In a separate terminal, "cargo run Party1_Settings" 
(8) after exuctution, weights will be generated.
(9) edit the file path in accuracy_test.py to the weights location and x y test files
(10) run accuracy_test.py with 'python accuracy_test.py <fold number>'
	- Note, the command line argument may be ignored if filepaths are completely specified



To run versions prior to 8/23/2019
(1) in idash2019_rust, cargo build
(2) ensure 'ti' is set to true in Settings.toml
(3) in one terminal, cargo run
(4) ensure 'ti' is set to false in Settings.toml
(5) update values in Settings.toml for party0
(6) in another terminal, cargo run
(7) update values in Settings.toml for party1
(8) in another terminal, cargo run
(9) after exuctution, weights will be generated.
(10) edit the file path in accuracy_test.py to the weights location and x y test files
(11) run accuracy_test.py
